Interactive Brokers Integration Guide
Risk Warning: Live trading involves real financial risk. Singularity's bot uses paper trading internally — connecting to IB means the bot's signals will be submitted as real orders to your broker. Always test with paper trading first. You are responsible for all trades executed through your account.

Overview

Singularity connects to Interactive Brokers via the IB API. You run IB Gateway or Trader Workstation on your local machine, and a small Python bridge script translates Singularity's signals into IB orders. This keeps your account credentials local — they never touch Singularity's servers.

Architecture

Singularity Bot (Browser)
    ↕ WebSocket / localStorage signals
Python Bridge Script (your machine)
    ↕ IB API (ib_insync / ibapi)
IB Gateway or TWS (your machine)
    ↕ IB Servers
Your Brokerage Account (Interactive Brokers)

Step-by-Step Setup

1
Install Interactive Brokers software
Download IB Gateway (recommended — lightweight, no GUI) or Trader Workstation (TWS) from interactivebrokers.com. Log in with your IB credentials. Enable API connections: Configuration → API → Settings → Enable ActiveX and Socket Clients. Set Socket port to 7497 (paper) or 7496 (live).
2
Install the Python bridge
Install dependencies on your machine:
pip install ib_insync flask flask-cors
Download the Singularity bridge script from your dev panel. It runs a small Flask server on port 8765 that Singularity's bot connects to via WebSocket.
3
Run the bridge script
python singularity_ib_bridge.py --host 127.0.0.1 --ib-port 7497 --account U1234567
The bridge will connect to IB Gateway, authenticate, and expose a local API that Singularity reads. You should see: ✓ Connected to IB · Account: U1234567
4
Connect Singularity
In the IB Live Dashboard panel above, enter your host (127.0.0.1), port (7497 for paper, 7496 for live), and account ID. Click Connect. Singularity will handshake with the bridge and start streaming your account data in real time.
5
Configure bot for live execution
In the Setup panel, set Trade Frequency and Max Positions. The bot will use your configured Risk Per Trade (%) against your real account balance. Set a conservative daily loss limit — the bot will stop sending orders when it's hit. Start with paper trading (port 7497) for at least 2 weeks before going live.
6
Monitor the live dashboard
The IB Live Dashboard updates in real time showing: open positions, unrealized P&L, today's closed trades, win rate, average hold time, and account NAV. You can halt all bot activity with the ■ Stop All button in the nav bar at any time.

Bridge Script (Python — save as singularity_ib_bridge.py)

from ib_insync import *
from flask import Flask, jsonify, request
from flask_cors import CORS
import threading, json, argparse

app = Flask(__name__)
CORS(app)
ib = IB()
ACCOUNT = ''

def connect(host, port, account):
    global ACCOUNT
    ACCOUNT = account
    ib.connect(host, port, clientId=1)

@app.route('/status')
def status():
    return jsonify({'connected': ib.isConnected(), 'account': ACCOUNT})

@app.route('/account')
def account():
    vals = {v.tag: v.value for v in ib.accountValues() if v.account == ACCOUNT}
    return jsonify({'nav': vals.get('NetLiquidation','0'),
                    'pnl_day': vals.get('DayTradesRemainingT+0','0')})

@app.route('/positions')
def positions():
    pos = [{'symbol': p.contract.symbol, 'qty': p.position,
            'avg_cost': p.avgCost, 'mkt_price': p.marketPrice(),
            'unreal_pnl': p.unrealizedPNL} for p in ib.positions()]
    return jsonify(pos)

@app.route('/order', methods=['POST'])
def place_order():
    data = request.json
    contract = Stock(data['symbol'], 'SMART', 'USD')
    action = 'BUY' if data['side'] == 'buy' else 'SELL'
    order = MarketOrder(action, data['qty'])
    trade = ib.placeOrder(contract, order)
    return jsonify({'order_id': trade.order.orderId, 'status': 'submitted'})

if __name__ == '__main__':
    parser = argparse.ArgumentParser()
    parser.add_argument('--host', default='127.0.0.1')
    parser.add_argument('--ib-port', type=int, default=7497)
    parser.add_argument('--account', required=True)
    args = parser.parse_args()
    threading.Thread(target=connect, args=(args.host, args.ib_port, args.account)).start()
    app.run(port=8765)

Important Notes

• IBKR paper trading accounts are free and available to all IBKR users. Always validate the bot there first.

• The bridge runs on your local machine — your credentials never leave it.

• Market orders are used for simplicity. For production use, consider limit orders and implement slippage controls.

• Interactive Brokers has a minimum account size of $10,000 for margin accounts. Cash accounts have no minimum.

• The bot's daily loss limit in Singularity is independent of IBKR's own risk controls — set both for double protection.
