Methodology

Built to professional
standards

01
No Toy Models
Every model is production-grade. GARCH uses maximum likelihood estimation with multiple restarts. HMM uses Baum-Welch EM. The Heston model uses characteristic function integration — exact pricing, no Monte Carlo noise.
02
Real Live Data
All price data, options chains, and Treasury yields are pulled live via yfinance. No hardcoded prices. The yield curve bootstraps actual par rates. Options IVs are computed from live bid/ask midpoints.
03
Numerically Robust
float32 arrays for memory efficiency. Ledoit-Wolf shrinkage for covariance estimation. Nearest positive-definite matrix projection. Brent's method for IV inversion. All NaN/Inf values sanitized before JSON transport.
04
Memory Safe
Hard caps on simulation counts (100k paths). Step-by-step GBM to avoid chunk allocation. One-model-at-a-time worker with explicit del and gc.collect() between models. Payload size capped at ~1MB via histogram bins.
05
Academic Rigor
Each model cites its primary source: Hagan (2002) for SABR, Andersen (2007) for Heston QE, Almgren-Chriss (2001) for optimal execution, Merton (1974) for structural credit, Baum-Welch (1970) for HMM.
06
Extensible Architecture
Each project is fully self-contained in its own folder. The engine layer has zero HTTP dependencies. New models can be added to any engine without touching the Flask layer or frontend.
